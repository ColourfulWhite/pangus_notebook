GFM 警告

支持 Github 的 GFM 警告，Vuepress-Theme-Hope 重写了 GFM 警告的样式，与提示容器保持一致。

这两者区别在于直接在 Github 查看 Markdown 的时候，后者可以渲染成 GFM 的警告样式。

选项卡

选项卡包装在 tabs 容器中。

可以在 tabs 容器中添加一个 id 后缀，该后缀将用作选项卡 id。 所有具有相同 id 的选项卡将共享相同的切换事件。

<!-- 👇 这里，fruit 将用作 id，它是可选的 -->

::: tabs#fruit

<!-- 选项卡内容 -->

:::

在这个容器内，使用 @tab 标记来标记和分隔选项卡内容。

在 @tab 标记后，可以添加文本 :active 默认激活选项卡，之后的文本将被解析为选项卡标题。

::: tabs

@tab 标题 1

<!-- tab 1 内容 -->

@tab 标题 2

<!-- tab 2 内容 -->

<!-- 👇 tab 3 将会被默认激活 -->

@tab:active 标题 3

<!-- tab 3 内容 -->

:::

默认情况下，标题将用作选项卡的值，但可以使用 id 后缀覆盖它。

::: tabs

<!-- 👇 此处，选项卡 1 的标题“标题 1”将用作值。 -->

@tab 标题 1

<!-- tab 1 内容 -->

<!-- 👇 这里，tab 2 的标题将是 “标题 2”，并且它会使用 “值 2” 作为选项卡的值-->

@tab 标题 2#值 2

<!-- tab 2 内容 -->

:::

可以在每个选项卡中使用 Vue 语法和组件，并且可以访问 value 和 isActive，表示选项卡的绑定值和选项卡是否处于激活状态。

如果想让一些选项卡组一起切换，可以使用相同的选项卡 ID 来绑定它们。针对每个选项卡 ID 的选择会被存储并进行持久化。

脚注

  • 在 Markdown 中使用 [^锚点文字] 来定义脚注。

  • 在之后的任何位置使用 [^锚点文字]: ... 来描述脚注内容。

  • 如果脚注包含多个段落，其后的段落应当保持双层缩进。

例子

脚注 2 链接[2]。

脚注 2 链接[^second]。

行内的脚注^[行内脚注文本] 定义。

重复的页脚定义[^second]。

[^first]: 脚注 **可以包含特殊标记**

导入文件的 Docs。

Git

Notebook 使用 Git 进行版本控制，commit 的 message 按统一的格式撰写。

基本格式

  • header：<type>: <subject>
  • body：<body>
  • footer：<footer>

每部分使用一个空行隔开。

header 部分不能超过 50 个字符。

body 和 footer 每行不能超过 72 个字符。

一个完整的 Git Message 格式如下：

<type>: <subject>
<BLANK LINE>
<body>
<BLANL LINE>
<footer>

header

type 字段是必须的。

可选：

  • feat: 新增的内容

  • fix: 改动的内容

  • update: 更新主题的版本

subject 字段是必须的。

subject 的要求如下：

  • 简短，保证整个 header 部分不超过 50 个字符
  • 使用祈使句，第一人称现在时
  • 以小写字母开头
  • 结尾不加句号或其他标点符号

body

body 部分是可选的。

body 部分使用第一人称现在时间。e.g. 使用 change 而不是 changed 或 changes

保证每行不超过 72 个字符，如果超过手动换行。

footer

footer 部分是可选的，用于说明包含 BREAKING CHANGE ，参考链接，工单单号或关闭 issue 等。

  • BREAKING CHANGE

    如果当前提交包含不兼容的提交，需要在 footer 中包含 BREAKING CHANGE: <description> ，同时需要在 type 字段后面添加一个 ! 。

  • 关闭 issue

    如果当前 Commit 是针对某个 issue 的，可以在 footer 中使用 Closes #<issudid> 关闭这个 issue。

    关闭一个 issue：

    Closes #1

    关闭多个 issue：

    Closes #1, #2, #3
